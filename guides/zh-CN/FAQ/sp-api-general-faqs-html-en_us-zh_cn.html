账户

我是否需要注册专业销售账户才能成为销售伙伴 API 开发者？

点击此处以展开回答。

是的，只有注册专业销售账户才能开发销售伙伴 API 或与之集成。个人账户不符合条件。您可以随时将您的账户升级为专业计划。可以从账户信息的“您的服务”下查看您的销售计划类型和商城信息。

是否必须拥有专业销售账户才能使用沙箱端点？

点击此处以展开回答。

是的，对 SP-API 进行沙箱调用与进行生产调用相同，区别在于您要将调用指向 SP-API 沙箱端点。调用沙箱端点会返回所有销售伙伴 API 的静态模拟响应。沙箱端点允许您在不影响生产数据或触发真实事件的情况下测试应用程序。

更新开发者资料是否会影响现有的应用程序或职权？

点击此处以展开回答。

不会。当您提交更改以更新开发者资料时，将创建一个新 case。根据您收到的 case 反馈，您需要在回复中提供的截止日期之前更新/修改您的个人资料。

更新开发者资料后，完成审查需要多长时间？

点击此处以展开回答。

审查通常需要至少 2 周。如果您没有受限 (PII) 职权，并且是首次申请这些职权，则审查花费的时间会更长。亚马逊需要使用最新 ADP 审查受限 (PII) 职权，ADP 会经常更新。

授权

为什么我无法成功调用授权 API，或者为什么 getAuthorizationCode 返回“未找到 MWS 授权”？

点击此处以展开回答。

如果您使用尚未发布的 SP-API 应用程序或混合应用程序向授权 API 发出请求，则会返回此错误。请注意向授权 API 发出有效请求的以下先决条件：

  • 已发布的 MWS 应用程序
  • 已发布的 MWS 应用程序已转换为混合应用程序
  • 混合应用程序还发布到亚马逊销售伙伴应用商店
  • MWS 的所有开发者 ID 都已添加到混合应用程序中，卖家也已在 MWS 中授权了该应用程序。

为什么我需要在迁移期间使用授权 API？

点击此处以展开回答。

您可以使用授权 API 实现错误处理，以便在迁移期间 SP-API 代码出错时回退到 MWS 代码。

测试授权工作流程时，为什么“确认”按钮不起作用？

点击此处以展开回答。

为了使授权工作流程在亚马逊销售伙伴应用商店和网站工作流程中都能正常运行，SP-API 应用程序必须包含 OAuth 登录 URI 和重定向 URI 的详细信息。未能为这两个字段中的任一个添加信息均会导致“确认”按钮不起作用。

在混合应用程序的 OAuth 工作流程中，为什么 spapi_oauth_code 没有返回 MWS 身份验证令牌？

点击此处以展开回答。

对于自我授权用例（即，使用 OAuth 工作流程授权您自己的卖家账户），不会返回 MWS 身份验证令牌。

我在注册我的应用程序时，要在 OAuth 信息中添加哪些登录和重定向 URI 值？

点击此处以展开回答。

OAuth 登录 URI 定义了网站在授权期间的登录页面。此 URI 是在卖家同意授权您的应用程序后显示的。

OAuth 重定向 URI 用于在授权后将浏览器重定向到您的应用程序。您可以通过在 OAuth URL 中包含 redirect_uri 参数来指定多个重定向。如果未指定 redirect_uri 参数，它将默认为您定义的第一个 OAuth 重定向 URI。

注意： OAuth 重定向 URI 不支持 localhost: URI。必须同时指定 OAuth 登录 URI 和 OAuth 重定向 URI，才能成功将其保存到您的应用程序注册中。

刷新令牌何时到期？

私人开发者点击此处以展开回答。 刷新令牌的有效期很长。
公共开发者点击此处以展开回答。 刷新令牌在一年后到期。

我们可否通过 SP-API 获取刷新令牌的到期日期？

点击此处以展开回答。

不能。但是，我们将在到期前 30 天向卖家发送一封电子邮件，提醒他们重新授权其 SP-API 应用程序。

如何获得卖家的应用程序发布授权？

点击此处以展开回答。

在 2022 年 12 月 31 日之前，用户可以使用授权 API 将授权令牌从 MWS 转换为 SP-API。

12 月 31 日之后，您需要通过销售伙伴应用商店授权工作流程或网站授权工作流程实现 OAuth 授权。

发布和编辑应用程序

如何发布应用程序？ 该过程需要多长时间？

点击此处以展开回答。

发布应用程序的过程：

  1. 访问开发者平台。
  2. 要更新现有应用程序，请从要发布的应用程序的下拉列表编辑应用程序中选择创建商品信息。
  3. 输入应用程序信息。
  4. 提交您的应用程序。

注意： 完成整个过程并发布应用程序通常需要两周以上的时间。

创建 SP-API 应用程序或将 MWS 转换为混合应用程序后，我能否更改 IAM ARN？

点击此处以展开回答。

可以。当您在开发者平台页面（在卖家平台或供应商平台中）编辑应用程序信息时，可以将 IAM ARN 替换为新值。旧 IAM ARN 和新 IAM ARN 的有效期均为 14 天，这使您有时间更新您对新 IAM ARN 的授权工作流程。14 天后，旧的 IAM ARN 过期。

我想更新已经发布的应用程序。如果我要提交包含更新信息的应用程序，这是否会影响现有的应用程序？

点击此处以展开回答。

不会。现有应用程序在发布过程中不会受到影响。更改获得批准并发布后，新版本将显示给客户。

一般问题

如何使用多个重定向 URI？

点击此处以展开回答。

当销售伙伴登录您的网站（销售伙伴应用商店授权工作流程中的第 3 步）时，您的应用程序会将亚马逊回调 URI 加载到浏览器中，并添加重定向 URI、amazon_state 参数和 state 参数。

要为应用程序使用多个重定向 URI，必须在注册应用程序时添加 OAuth 重定向 URI。请注意，必须同时指定 OAuth 登录 URI 和 OAuth 重定向 URI 值，才能成功将其保存到您的应用程序注册中。OAuth 重定向 URI 不接受 localhost: URI。

如果为应用程序配置了多个重定向 URI，请在销售伙伴应用商店授权工作流程的第 3 步中传递要用作查询参数的重定向 URI。如果您不包括 redirect_uri 参数，则默认为您在注册应用程序时指定的第一个 OAuth 重定向 URI。

亚马逊是否为 Ruby/PHP 提供客户端库？

点击此处以展开回答。

不。亚马逊不支持 Ruby/PHP 的客户端库。

如果同一 API 有多个版本，我应该使用哪个版本？

点击此处以展开回答。 我们建议使用最新版本。最新版本消除了较早的错误，而较旧版本则相应地被弃用。
