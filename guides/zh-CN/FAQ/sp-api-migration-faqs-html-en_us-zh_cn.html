本常见问题解答涵盖了开发者在从亚马逊商城网络服务 (MWS) 迁移到销售伙伴 API (SP-API) 时可能遇到的问题。回答可能会有所不同，具体取决于您的应用程序以及是私有还是公共应用程序。

一般迁移主题

亚马逊 MWS 什么时候会停用？

点击此处以展开回答。

2024 年 3 月 31 日之后，MWS 将不再可用。亚马逊已选择分阶段弃用亚马逊 MWS，并且我们已经公开宣布了第三方公共应用程序开发者和私有应用程序开发者的迁移截止日期。

为什么亚马逊用 SP-API 取代 MWS？

点击此处以展开回答。

较亚马逊 MWS 相比，SP-API 为所有开发者进行了众多改进，例如基于 JSON 的现代化 REST API 设计标准、使用“使用亚马逊账户登录”进行的 OAuth2.0 销售伙伴授权以及 MWS 中最受欢迎的功能——测试端点。除了具备以往亚马逊 MWS 提供的所有旧版功能和面向卖家的全新 API 功能之外，SP-API 还将使用相同的 API 控制平面和授权机制为首个面向亚马逊供应商的 API 提供支持。日后，所有的全新 API 功能都将内置于 SP-API 中，在 MWS 被弃用之前，对 MWS 的更新仅用于支持业务关键型变更。将所有开发者与 SP-API 集成，可以为销售伙伴和开发者提供最佳体验。

可以在哪里了解有关 SP-API 的更多信息？

点击此处以展开回答。

有关 SP-API 的更多信息，请参阅销售伙伴 API 开发者指南。

是否必须从亚马逊 MWS 迁移到 SP-API？

私人开发者点击此处以展开回答。

是。对于拥有亚马逊 MWS 开发者 ID 和访问密钥并使用 MWS 卖家配送、订单或报告 API 部分的私人 MWS 开发者，必须在 2023 年 8 月 31 日之前将这些部分迁移到 SP-API，以防止业务中断。此外，使用所有其他 MWS API 部分的私人 MWS 开发者必须在 2024 年 3 月 31 日之前完全迁移到 SP-API。

如果私人开发者决定不与 SP-API 集成，则可以授权来自亚马逊销售伙伴应用商店的第三方应用程序，用这些应用程序取代他们的亚马逊 MWS 私有集成。

公共开发者点击此处以展开回答。

是。亚马逊 MWS 弃用用例的第一阶段已于 2021 年 4 月 6 日开始，针对商城应用程序商店中的销售伙伴授权的所有公共应用程序的订单、报告和卖家自配送部分。为了防止业务中断，您必须在 2022 年 7 月 31 日之前进行迁移。

此外，对于销售伙伴授权的所有使用 MWS 商品 API 的公共应用程序，都必须在 2022 年 9 月 30 日之前迁移到 SP-API。

对于销售伙伴授权的所有使用所有其他 MWS API 部分的公共应用程序，则必须在 2022 年 12 月 31 日之前完全迁移到 SP-API。

什么是私人开发者和公共开发者？

点击此处以展开回答。 私人开发者开发将自己的公司与亚马逊 API 集成的应用程序。

公共开发者开发并提供其他公司使用的公开可用的应用程序。

什么是私有应用程序和公共应用程序？

点击此处以展开回答。 私有应用程序仅对您的组织可用且经过自我授权。

公共应用程序是其他公司使用的公开可用的应用程序。

如何知道我是私人 MWS 开发者还是有权访问 MWS？

点击此处以展开回答。 如果您使用 MWS 来支持您的业务自动化需求，则您是私人开发者，并且已经为您的账户分配 MWS 开发者 ID 和 MWS 密钥。使用以下过程查看您的凭证：
  1. 以主要账户持有人身份登录卖家平台上的亚马逊销售账户。选择设置，然后选择用户权限。
  2. 选择亚马逊 MWS 开发者访问密钥，然后选择访问开发者凭证。
  3. 在开发者平台页面上，选择 MWS 凭证，然后选择查看。

什么是 亚马逊销售伙伴应用商店？

点击此处以展开回答。

亚马逊销售伙伴应用商店是一站式商店，卖家可以在这里找到可信的应用程序，以帮助发展和管理您的业务。销售伙伴应用商店将卖家与亚马逊批准的第三方软件合作伙伴联系起来，从自动定价到发布工具，这些合作伙伴可以帮助您在线启动和发展业务。许多应用程序都专注于自动执行曾经耗时且繁琐的日常任务，如商品研究、填写税务表格或构建定制报告。

我是私有供应商应用程序的私人开发者，我需要迁移吗？

点击此处以展开回答。 供应商直接加入了 SP-API。除非您有权访问其他用例的 MWS，否则无需迁移。

哪些迁移截止日期适用于我的私有应用程序？

私人开发者点击此处以展开回答。

如果您拥有调用卖家配送 (MFN)、订单或报告 API 部分的私有亚马逊 MWS 开发者 ID 和访问密钥，则必须创建并运行私有销售伙伴应用程序来集成这些部分。上述操作必须在 2023 年 8 月 31 日之前完成。

注意： 如果您有需要受限数据的混合或基于 SP-API 的私有应用程序，则必须在 2023 年 8 月 31 日之前更新应用程序，以通过令牌 API 使用受限数据令牌 (RDT)。您可以在此处查看受影响的操作列表。

如果您拥有使用任何其他 MWS API 部分的私有亚马逊 MWS 开发者 ID 和访问密钥，则必须在 2024 年 3 月 31 日之前创建并运行与 SP-API 集成的私有销售伙伴应用程序。这包括使用以下 MWS API 部分的应用程序：

* 上传数据 API
* 财务 API
* 亚马逊物流入库货件 API
* 配送库存 API
* 亚马逊物流出库货件 API
* 商品 API
* 销售指导 API
* 卖家 API
* 订阅 API
* 推送通知架构

如果私人开发者决定不与 SP-API 集成，则可以授权来自亚马逊销售伙伴应用商店的第三方应用程序，用这些应用程序取代他们的亚马逊 MWS 私有集成。

注意： 2022 年，我们将迁移工作重点放在由亚马逊销售伙伴授权的第三方公共应用程序上，这些应用程序也可能会发布到亚马逊销售伙伴应用商店中。有关第三方迁移的更多信息，可以在本公告中找到。

公共开发者点击此处以展开回答。

2022 年，我们将迁移工作重点放在由亚马逊销售伙伴授权的第三方公共应用程序上，这些应用程序也可能会发布到亚马逊销售伙伴应用商店中。我们将通过电子邮件和卖家平台 case 向所有受此影响的 MWS 公共 MWS 开发者直接发送通信，这将有助于确认贵组织必须采取的行动。以下是适用于需要迁移到 SP-API 的公共 MWS 开发者的条件：

  • 如果您有使用卖家配送网络 (MFN)、订单或报告 API 的基于亚马逊 MWS 的公共应用程序，则必须在 2022 年 7 月 31 日之前更新您的应用程序，以便将这些部分集成到 SP-API。

  • 如果您有需要受限数据的混合或基于 SP-API 的公共应用程序，则必须在 2022 年 7 月 31 日之前更新应用程序，以通过令牌 API 使用受限数据令牌 (RDT)。您可以查看受影响的操作列表。

  • 如果您有使用商品 API 的基于亚马逊 MWS 的公共应用程序，则必须在 2022 年 9 月 30 日之前更新您的应用程序，以便与 SP-API 中的目录商品和商品信息集成。

  • 如果您有使用任何其他 MWS API 部分的基于亚马逊 MWS 的公共应用程序，则必须在 2022 年 12 月 31 日之前更新您的应用程序，以便与 SP-API 完全集成。这包括使用以下 MWS API 部分的应用程序：

    • 上传数据 API
    • 财务 API
    • 亚马逊物流入库货件 API
    • 配送库存 API
    • 亚马逊物流出库货件 API
    • 销售指导 API
    • 卖家 API
    • 订阅 API
    • 推送通知架构

私人 MWS 开发者： 有关私人开发者迁移的信息，请参阅亚马逊商城网络服务 (MWS) 的生命周期终止日期

“将停止为亚马逊 MWS 订单、报告和卖家配送网络 (MFN) 部分提供支持”是什么意思？

私人开发者点击此处以展开回答。

停止提供支持意味着，在 2023 年 8 月 31 日之后，使用亚马逊 MWS 指定部分的私有亚马逊 MWS 开发者 ID 和访问密钥的 API 调用将不会成功。

2023 年 8 月 31 日及之后，对所有其他 Amazon MWS 部分（操作和路径）的调用将继续保持成功调用，就如同今天一样。

公共开发者点击此处以展开回答。

停止提供支持意味着，在 2022 年 7 月 31 日之后，使用亚马逊 MWS 指定部分的公共应用程序调用 API 将不会成功。

2022 年 7 月 31 日及之后，对所有其他 Amazon MWS 部分（操作和路径）的调用将继续保持成功调用，就如同今天一样。

我是私有亚马逊 MWS 应用程序的销售伙伴，想找一个第三方应用程序进行授权，而不是迁移。如何找到适合我的业务的应用程序？

点击此处以展开回答。

访问亚马逊销售伙伴应用商店，然后键入关键字，以根据名称或支持功能搜索应用程序。您还可以按支持的语言、支持的商城、定价类型和其他搜索筛选条件来细化搜索范围。在主页上，除了个性化应用程序建议外，您还会看到可供探索的分类列表。

一家外部开发公司构建了我们现有的 MWS 应用程序，但它们不再可用，而且我没有开发应用程序的经验。我该怎么做？

点击此处以展开回答。

如果您有权访问应用程序的原始代码文件，则可以与另一家外部开发公司签订合同，以帮助您将您的 MWS 应用程序迁移到 SP-API，或者请求他们帮助您创建一个使用 SP-API 的新应用程序。

此外，您还可以选择寻找第三方应用程序解决方案来取代您的私有亚马逊 MWS 集成。您必须在发布的截止日期之前根据您的 MWS API 使用情况采取措施，以避免业务中断。我们鼓励您访问亚马逊销售伙伴应用商店，查看超过 2000 款经过仔细检查的应用程序，这些应用程序提供了满足您业务需求的各种自动化解决方案。

如果我开始迁移流程，我的应用程序支持的日常工作流程会受到任何干扰吗？

点击此处以展开回答。

迁移期间，您的应用程序将继续照常使用 MWS。在发布的截止日期之前，或者在您确认已完成迁移/替换 MWS 应用程序功能的必要操作之前（以先发生者为准），我们不会从您的账户中删除 MWS 访问权限。

2023 年 8 月 31 日之后，所有私人开发者将无法访问 MWS 卖家配送网络 (MFN)、订单和报告 API 部分。

2024 年 3 月 31 日之后，所有应用程序将无法使用所有其他 MWS API 部分。

迁移到 SP-API 的过程对我来说似乎太复杂了，我无法完成。如果我无法迁移或不想迁移，我有哪些选择？

点击此处以展开回答。

如果您决定不创建和拥有私有销售伙伴应用程序，或者不想将应用程序迁移到 SP-API，则可以选择寻找第三方应用程序解决方案来取代您当前的业务自动化服务。我们鼓励您访问亚马逊销售伙伴应用商店，查看超过 2000 款经过仔细检查的应用程序，这些应用程序提供了满足您业务需求的各种自动化解决方案。

您可以采纳的另一个选项是在线搜索提供签约开发服务的外部公司，以帮助您将应用程序从 MWS 迁移到 SP-API，或者创建新的私有 SP-API 应用程序。

应用程序列在“管理您的应用程序”部分中。我需要采取什么措施？

点击此处以展开回答。 这部分中显示的应用程序是第三方公共应用程序，这表示您已授权访问您的销售信息以支持您的业务自动化需求。这些应用程序由负责将应用程序迁移到 SP-API 的第三方公共开发者拥有和开发。作为应用程序的用户，您无需对这些应用程序采取任何措施。

我使用 MWS 访问商品信息加载工具或亚马逊卖家运输实用工具 (AMTU) 等应用程序，我需要迁移吗？

点击此处以展开回答。

在 2023 年 8 月 31 日之前，AMTU/商品信息加载工具将保留对 MWS 所有功能的访问权限。在此日期之后，AMTU 工具将被弃用。更多信息将于 AMTU 可用时发布在 AMTU 帮助页面上。

注意： 有关 AMTU 用户提交汽车售后市场目录交换标准 (ACES) 配件上传数据的重要信息： 2023 年 4 月 1 日或之前，通过 AMTU 提交配件数据的汽车销售伙伴和第三方数据提供商需要迁移到新工具，以便提交 ACES 上传数据。请按照提供和添加配件数据页面上的说明进行操作。

亚马逊 Pay 开发者是否受迁移影响？

点击此处以展开回答。 使用 MWS 的亚马逊 Pay 开发者将受到 MWS 生命周期终止日期的影响，此日期为： 2024 年 3 月 31 日。有关亚马逊 Pay 开发者必须采取哪些措施的更多详细信息，我们将于今年晚些时候单独通知大家。

 

账户和访问主题

我是一名公共开发者，回复了来自 dev-reg-notice@ 的电子邮件并确认我已完成到 SP-API 的迁移。接下来会发生什么？

点击此处以展开回答。

如果您有混合 (MWS/SP-API) 应用程序：

  1. 我们会将您的混合 SP-API 应用程序转换为仅限 SP-API 的应用程序。注意： 您的应用程序必须处于“已发布”状态。
  2. 在您的应用程序仅转换为 SP-API 后，我们将阻止您接受新卖家 MWS 授权。注意： 如果您的应用程序类型仍为“混合”，则不会执行此操作。
  3. 我们将移除您对所有 MWS API 部分的访问权限。

注意： 或者，您可以按照将公共亚马逊 MWS 应用程序转换为销售伙伴 API 应用程序教程，将您的应用程序从混合应用程序转换为 SP-API。需要等待最多 15 个工作日才能发布此更改。

如果您有 MWS 应用程序：

  1. 我们将阻止您接受新的卖家 MWS 授权。
  2. 我们将移除您对所有 MWS API 部分的访问权限。

我们将在对您的 MWS 访问权限进行任何更改前 2 天通知您，更改完成后将向您发送确认邮件。

我的卖家账户已被停用。我该如何继续迁移？

点击此处以展开回答。

请通过获取支持联系销售伙伴支持团队，以请求重新激活您的账户。

如何申请 SP-API 访问权限或更新我有权访问的 SP-API 职权？

私人开发者点击此处以展开回答。

如果您已经获得 MWS 的访问权限，则无需在 SP-API 中申请访问权限，除非您想在 SP-API 中获得新职权。

在 MWS 中拥有有效 MWS 密钥并分配了访问职权的所有现有 MWS 开发者均已有权访问 SP-API 中的等效职权，这样您就可以开始迁移了。

如果您想要为账户添加新的 SP-API 职权，则可以随时访问卖家平台 URL 页面，然后选择您所在区域的 URL，以便提交您的 SP-API 开发者资料更新内容。选择应用商店，然后依次选择开发应用程序和查看个人资料，最后选择注册，以将职权添加到您的账户中。

您还可以通过卖家大学的视频指南了解有关 SP-API 职权的更多信息，并找到更多信息以开始集成。

公共开发者点击此处以展开回答。

在开发者注册页面提交或更新您的开发者资料。

您还可以通过卖家大学的视频指南了解有关 SP-API 职权的更多信息，并找到更多信息以开始集成。

我在我的卖家平台账户中未看到编辑应用程序、开发应用程序或注册应用程序选项，因此无法选择它们。我该怎么做？

点击此处以展开回答。 确保您登录的卖家平台账户是与您的 MWS/SP-API 应用程序关联的账户。这应该是贵组织向其提交开发者注册和评估表格/开发者资料详细信息的账户。

您还应该以该账户的主要用户身份登录。账户的次级用户可能与主要用户确定的职权不同。

您还必须确认您的账户目前是专业销售账户。只有专业销售账户才能注册开发 MWS、SP-API 或与 MWS、SP-API 集成。个人级别的账户不符合条件。导航到您的账户信息页面，在账户信息中选择您的服务，以便查看您的销售计划类型和商城信息。

我无法继续迁移，因为我的开发者资料 case 或亚马逊销售伙伴应用商店商品信息 case 仍在等待审查。

点击此处以展开回答。

我们的开发者支持团队可能需要几天时间才能在完成评估后做出回应，具体取决于您在开发者资料中提交的信息。继续关注 case 以了解最新情况。

公共开发者注意事项：如果您最近提交了新的销售伙伴应用商店商品信息，则在我们的开发者支持团队审查这些详细信息并发布之前，您可能无法进行其他更新。

我知道我需要设置一个 AWS 账户，但我需要为这个账户付费吗？

点击此处以展开回答。

AWS 身份与访问管理 (IAM) 服务不收取任何相关费用。AWS 账户设置要求您输入付款方式，但是使用 IAM 不会产生任何费用。但是，如果您打算使用任何其他 AWS 资源，则这些资源将按照 AWS 定价收费。

我正在尝试更新我的亚马逊销售伙伴应用商店商品信息，但我的应用程序显示为已禁止。可以将其重新激活吗？

点击此处以展开回答。

您可以通过从下拉菜单中选择编辑应用程序来编辑处于已禁止状态的应用程序。提交更新后，您的应用程序状态将更改为编辑待审查。

如果我是私人开发者，如何授权第三方应用程序？

点击此处以展开回答。

无需登录即可使用亚马逊销售伙伴应用商店，方便潜在卖家查找解决方案。当前亚马逊卖家还可以在卖家平台导航栏的顶部菜单中找到亚马逊销售伙伴应用商店，选择查找应用程序即可进入应用商店。

访问亚马逊销售伙伴应用商店，然后键入关键字，以根据名称或支持功能搜索应用程序。您还可以按平均买家评论评分、支持的语言、支持的商城、定价类型和其他搜索筛选条件来细化搜索范围。在销售伙伴应用商店主页上，除了您的企业可能觉得有用的推荐应用程序外，您还可以看到一系列可供探索的分类。

您可以选择这些应用程序的商品详情页面，进一步了解应用程序服务所提供的功能及其描述。

找到适合您业务的应用程序后，请选择应用程序商品详情页面右上角的立即授权。随后您将被定向到必须提交的确认页面，允许第三方应用程序访问您的销售伙伴账户和相关数据，以满足您的业务自动化需求。

您可以随时在卖家平台账户的管理您的应用程序页面取消对应用程序的授权。

如果我是私人开发者，那么我在 SP-API 中有开发者 ID 和访问密钥吗？

如果我将我的公共 MWS 应用程序转换为同时使用亚马逊 MWS 和 SP API 的混合 SP-API 应用程序，我的应用程序能继续进行 MWS 调用吗？

点击此处以展开回答。

是的，亚马逊 MWS 将继续使用混合应用程序运行。发布混合应用程序后，MWS 调用和授权将继续运行；但是，授权工作流程将更改为销售伙伴从亚马逊销售伙伴应用商店启动的 OAuth 授权工作流程。只有在混合应用程序发布后，才能使用销售伙伴 API 将现有的亚马逊 MWS 授权迁移到 SP-API 进行授权。有关更多信息，请参阅授权 API 用例指南。
