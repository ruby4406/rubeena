与调用授权 API 相关的常见错误

为什么我无法成功调用授权 API，或者为什么 getAuthorizationCode 返回“未找到 MWS 授权”？

点击此处以展开回答。

如果您使用的是正确的令牌，但仍收到此错误，则您可能是使用了尚未发布的 SP-API 应用程序或混合应用程序来调用授权 API。请注意向授权 API 发出有效请求的以下先决条件：

  • 必须发布您的 MWS 应用程序。
  • 已发布的 MWS 应用程序必须转换为混合 SP-API 应用程序。
  • 混合 SP-API 应用程序必须发布到亚马逊销售伙伴应用商店。
  • 必须将所有 MWS 开发者 ID 添加到混合 SP-API 应用程序中，并且卖家必须授权您在 MWS 中使用的 SP-API 应用程序或混合 SP-API 应用程序。

为什么我在运行授权 API 时收到错误“缺少身份验证令牌”？

点击此处以展开回答。 如果您在 API 请求中使用 LWA 凭证并且缺少 AWS 密钥，则会返回此错误。要解决此错误，请确保在您的 API 授权请求中添加 AWS `AccessKey`、`SecretKey` 和会话令牌（如果使用 STS）。此外，还必须在请求标头 `x-amz-access-token` 中使用以 `Atc|*************` 开头的访问令牌。

为什么在使用授权 API 时收到与应用程序 ID 无关的开发者 ID 错误？

点击此处以展开回答。

必须发布该应用程序。要发布您的应用程序，请按照以下步骤操作：

  1. 使用您用于注册成为开发者的凭据登录卖家平台。
  2. 在应用商店菜单中，选择开发应用程序。此时将显示开发者平台页面。
  3. 选择应用程序右侧的下拉菜单，然后选择“编辑应用程序”。
  4. 编辑该页面上要更新的值，并确保流程的三个部分中没有缺失任何信息，此三个部分为： 应用程序信息、定价和应用程序详细信息。
  5. 如果信息不完整，第三步结束时将显示“保存并退出”选项。返回并确保您已输入所有必填信息。
  6. 输入所有必填信息后，选择“保存并提交”以确认并发送要发布的应用程序。

完成此操作后，应用程序验证团队将清除该应用程序，并将自动创建一个 case。您将收到该应用程序将于 10 天内获得批准的通知，这将允许您为第三方发出调用。出于测试目的，您可以通过自授权调用为自己的账户发出调用。

如果问题仍然存在，请联系开发者支持团队。

运行授权 API 时，为什么会收到“应用程序未配置职权”错误？

点击此处以展开回答。 如果在发布时未将职权正确添加到混合应用程序或混合应用程序尚未完成发布，则会返回此错误。

要解决此错误，请执行以下操作：

  • 确保您的混合应用程序已完全发布。审查后，亚马逊将向您发送通知，确认您的应用程序已完全发布。
  • 检查 SP-API 职权是否已添加到您发布的混合应用程序中。

为什么我会收到“请求中包含的安全令牌无效”？

点击此处以展开回答。 验证您在请求中使用的 AWS 访问密钥和私有密钥对是否与您添加到应用程序的 IAM 实体相关联。如果您使用的是 IAM 职权，则需要向 AWS STS 发出请求，以请求临时 AWS 密钥并将这些密钥用于请求。

为什么我收到交易 API 调用的“处理中”消息？

点击此处以展开回答。

以下 API 操作支持交易状态：

  • 确认订单
  • 提交货件确认
  • 提交货件状态更新
  • 货件标签请求
  • 库存数据上传

如果交易状态为“处理中”，或者在 5 分钟后未更新为“失败”或“成功”，则表示交易未在我们的系统中成功完成。如果交易成功，则会出现“成功”状态，如果“失败”，则会提供错误代码。

您不能将此 API 用于“发票”消息，因为不支持对此类消息进行检查，并且状态将始终为“处理中”。

对于货件标签请求，如果交易失败且提供原因代码（例如是终端错误的“内部服务器错误”），则您必须创建“联系我们”case 以调查原因。这通常是由于操作限制而发生的。

为什么我会收到“提交发票”操作调用的错误消息？

点击此处以展开回答。

必须正确填写操作参数。在填写发出 API 调用的参数时，请考虑以下几点：

  • 发票编号必须是唯一的，不得重复使用（即使在一年后也是如此）。
  • 如果 API 发送的发票由于数据不正确而失败，但纸质发票的数据正确，则供应商应通过 API 使用相同发票 ID 下的正确数据对其进行更新。
  • 如果发票包含错误的数据（包括纸质和 API），则发票将被取消，并且应使用新的发票编号发送新发票。
  • 不应发送总金额为“0”的发票，因为这会导致发票失败。
  • 出于税务合规原因，亚马逊要求在地址段中提供完整的地址详细信息。这对于收票方来说尤其重要。对于此部分，亚马逊收款人系统要求精确匹配。
  • 以发票形式发送的付款条件应该就是与亚马逊买家商定的付款条件
  • 商品标识符应与在匹配采购订单时发送给供应商的订单商品标识符相匹配。发票总额应等于商品、费用和补贴的总和。
  • 每个行级别的总税额必须等于标题级别的总税额。
  • 发票总数量应等于所有商品的数量之和。每种不同的费用和补贴都必须在标题级别逐项列出。

如果其中一个参数缺失或填写不正确，这将导致 API 调用检索错误的详细信息。提交操作时，请务必提供所需的信息。

为什么我会收到 SPDC300 错误？

点击此处以展开回答。

此错误可能是由于使用某些 HTML 标签，尤其是标题标签 <h1><h2><h3><h4> 引起的。移除这些标题标签，仅使用文本编辑器中提供的标签。

此错误还可能表明由于缺乏活动，账户状态已变为休眠。您可以更新您的信用卡信息以重新激活账户。下次登录卖家平台时，您将被重定向到信用卡更新页面。您的账户将在信用卡更新大约 48 小时后恢复。

成功完成注册后，为什么我的 API 请求发生“未授权 - 访问被拒绝”错误（403 错误代码）？

点击此处以展开回答。

如果您收到 API 请求的“访问被拒绝”错误，请参阅对请求源的访问被拒绝的故障排除提示。

当您向销售伙伴 API 发送 HTTP 请求时，您需要对请求进行签名，以便亚马逊能够识别谁发送了这些请求。您可以使用 AWS 访问密钥签署请求，该密钥包括访问密钥 ID 和私有访问密钥。如果您使用 IAM 角色注册了应用程序，则必须使用 AWS 安全令牌服务 (AWS STS) 请求临时 AWS 访问密钥以签署您的请求。

在服务控制策略 (SCP) 中限制 IAM 职权或 IAM 用户的访问权限，可能会导致访问被拒绝错误。如果您使用服务控制策略 (SCP) 管理组织的访问权限，请确保 SCP 不限制成员账户的 IAM 用户或 IAM 职权。

有关使用 AWS STS 和可以帮助您实施的 AWS SDK 的更多信息，请参阅 AWS 文档中的请求临时安全凭证。

如果错误仍然存在，请记录并向亚马逊提交支持 case 以进一步解决问题。在支持 case 中包括以下详细信息：

  • 应用程序编号
  • 带时间戳的请求 ID
  • API 操作（指定是否为沙箱请求）
  • 收到的错误响应

当我在 Postman 中请求获取 LWA 访问令牌时，为什么会出现 500 错误？

点击此处以展开回答。

如果您收到 500 内部服务器错误，请检查 Content-Type 标头是否设置为 application/x-www-form-urlencoded，并将请求参数添加到正文而不是添加为查询参数。

我在向沙箱端点发出请求时为什么会出现 500 错误？

点击此处以展开回答。

SP-API 沙箱的工作原理与许多模拟框架相同，它采用模式匹配，在指定参数存在时返回指定的响应。开发者在发送与指定参数匹配的请求时会收到在 x- amazon-spds-sandbox-behaviors 对象中定义的响应。

如果发送到沙箱端点的请求与 x-amazon-spds-sandbox-behaviors 对象中的参数值不匹配，则您将在响应中收到“500 内部服务器错误”。您必须使用模型中指定的确切值发送请求。

如果 API 需要使用的任何参数未在 x-amazon-spds-sandbox-behaviors 对象中指定，则只要请求有效，沙箱都会提供响应，而不考虑请求中的参数值。

要了解有关对 SP-API 进行沙箱调用的更多信息，请参阅销售伙伴 API 开发者指南中的销售伙伴 API 沙箱部分。

为什么我会收到 MD1000 错误代码？

点击此处以展开回答。

OAuth 是其他卖家为授权您在应用商店中的应用程序而启动的授权流程。

如果包含 version=beta 参数，则工作流程将授权处于草稿状态的应用程序。如果不包括参数 version=beta，则该工作流程将在应用商店中授权该应用程序 ID 的已发布版本，否则将返回错误代码 “MD1000”。

为什么我在启动 OAuth 工作流程时收到 MD1000 错误代码？

点击此处以展开回答。

如果您的 SP-API 应用程序尚未发布，但 OAuth 工作流程指向生产工作流程，则会返回此错误。要解决此问题，请确认应用程序处于草稿阶段。如果处于草稿阶段，请将 version=beta 参数添加到构建的 OAuth 授权 URI 中。应用程序发布后，可以删除此参数。

为什么我会收到 MD5100 错误代码？

点击此处以展开回答。

MD5100 可能是由以下问题引起的：

  • OAuth 授权 URL 中存在错误。
  • 应用程序中缺少重定向链接。
  • 如果用户尝试在未添加到应用程序中的商城或区域中授权混合应用程序。
  • URL 中的片段。

试试以下解决方案能否解决 MD5100 错误：

  • 检查应用程序状态：
  • 如果应用程序处于草稿状态，请验证 URL 是否包含 version=beta。
  • 如果应用程序已发布，请确认 URL 中不包含 version=beta。如果 URL 中包含 version=beta，则针对草稿状态的应用程序而不是已发布状态的应用程序启动 OAuth 进程。
  • • 如果您的应用程序仅适用于亚马逊 MWS，并且您正在尝试自行授权该应用程序，请通过注册 SP-API 将该应用程序转换为混合应用程序。有关更多信息，请参阅自行授权。
  • 确认您已完成授权销售伙伴 API 应用程序的所有步骤。
  • 对于混合应用程序，请确保已为您运营的每个区域 (NA|EU|FE) 添加了 MWS 开发者 ID。您可以通过在开发者平台编辑应用程序来添加 MWS 开发者 ID。有关更多信息，请参阅创建混合应用程序。
  • 确认您的应用程序具有登录 URI 和重定向 URI。您可以在开发者平台编辑应用程序并更新登录 URI 和重定向 URI 详细信息，从而更新登录 URI 和重定向 URI。
  • 验证应用程序是否支持授权开发者进入的商城。在开发者平台中，为您的应用程序选择编辑商品信息选项，然后在“定价”部分选择应用程序应支持的商城。

如果在查看这些详细信息后问题仍然存在，请提交支持 case。

为什么我在使用带有特殊字符的 SKU 时收到错误消息？

import java.net.URLEncoder;
import java.nio.charset.StandardCharsets;
import java.io.UnsupportedEncodingException*;*


 // Method to encode a SKU using the UTF-8 encoding scheme
  private static String encodeSKU(String sSKU) {
    try {
      return URLEncoder.encode(sSKU, StandardCharsets.UTF_8.toString());
    } catch (UnsupportedEncodingException ex) {
      e*.*printStackTrace*();*
    }
}

有关更多信息，请参阅 URL 编码。
