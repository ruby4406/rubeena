フィードAPIとは何ですか？

Selling Partner API for Feeds（フィードAPI）を使用すると、Amazonに情報をアップロードするアプリケーションを作成できるので、出品者が出品ビジネスを管理するのに役立ちます。出品情報の作成、在庫と価格の管理、注文の確認など、さまざまなユースケースに対応するフィードがあります。使用可能なフィードタイプについては、フィードタイプ値を参照してください。

フィードを送信するためのワークフロー

フィードを送信するための手順の概要を以下に示します。

  1. createFeedDocumentオペレーションを呼び出し、送信するフィードのコンテンツタイプを指定します。

    Amazonから、feedDocumentId値と、フィードコンテンツをアップロードするURLが返されます。

  2. フィードドキュメントのコンテンツを前のステップのURLにアップロードします。

  3. createFeedオペレーションを呼び出します。inputFeedDocumentIdパラメーターを使用して、ステップ1のfeedDocumentId値を渡します。フィードを適用するマーケットプレイスと、関連するフィードオプションを指定します。

    AmazonからfeedId値が返されます。

  4. Amazon SQSキューを定期的にポーリングして FEED_PROCESSING_FINISHED通知イベントを確認します。このイベントは、フィード処理がCANCELLED（キャンセルされた）とき、DONE（完了した）、とき、FATAL（致命的）なときに情報を提供します。

    フィードがDONE状態に移行すると、AmazonからresultFeedDocumentId値の通知が返されます。

  5. getFeedDocumentオペレーションを呼び出します。feedDocumentIdパラメーターを使用して、前のステップのresultFeedDocumentId値を渡します。

    Amazonから、feedDocumentId値、フィード処理レポートをダウンロードするURL、圧縮アルゴリズムが返されます。

  6. フィード処理レポートをダウンロードします。

  7. フィード処理中に生成されたエラーがないか、フィード処理レポートを確認します。エラーがある場合は、それを修正して、修正済みフィードを送信します（ステップ1から開始）。エラーがない場合は、フィードの送信が成功しています。

フィードの送信について詳しくは、チュートリアル： フィードの送信をご覧ください。

用語

  • Amazon S3の署名付きURL： AWSセキュリティの認証情報やアクセス許可なしにオブジェクトをダウンロードできる、S3バケットのURL。オブジェクトが圧縮されている場合があり、その場合は、URLに加えて compressionAlgorithmが返されます。このURLは5分後に失効します。

チュートリアル： フィードの送信

このチュートリアルでは、フィードの送信方法、フィード処理ステータスの確認方法、フィードの送信が成功したことの確認方法について説明します。このチュートリアルに含まれているJavaコードサンプルで、フィードをアップロードする方法と、フィード処理概要レポートをダウンロードする方法を示します。サンプルコードに示される原則を使用して、他のプログラミング言語でアプリケーションを構築したり、他のHttpClientライブラリを使用したり、さまざまな形式のフィードをアップロードしたりできます。

前提条件

このチュートリアルを実行するには、以下が必要です。

  1. 送信するフィード。使用可能なフィードタイプについては、フィードタイプ値を参照してください。
  2. 呼び出しを行う出品者からの認可。詳細については、Selling Partner APIアプリケーションの認可をご覧ください。
  3. 使用可能なJava Development Kit（JDK）のインストール。

  1. 以下のパラメーターを渡して、createFeedDocumentオペレーションを呼び出します。

 はい

ステップ2.フィードを作成する

ステップ 3. フィードデータをアップロードするでアップロードできるフィードデータを作成します。

XMLフィード

XMLフィードを作成するには、3つのコアXSD（ベース、エンベロープ、ヘッダー）とカテゴリー固有のフィードを含める必要があります。

コアXSD：

  • ベース： フィード間の一貫性を高めるために使用します。他のすべてのXSDは、ベースXSDの要素とデータ型を参照します。
  • エンベロープ： 他のすべてのデータをメッセージレベルのプロトコルデータでラップするために使用します。ヘッダーと1つまたは複数のメッセージで構成されます。
  • ヘッダー： エンベロープXSDで、フィードまたはフィード内のメッセージに関連するユニバーサルデータを指定するために使用します。

カテゴリー別フィードのXSDへのリンクについては、セラーセントラルのヘルプでXSDsにアクセスし、Category XSDsセクションをご覧ください。

ステップ3.フィードデータをアップロードする

ステップ2.フィードを作成するで、ステップ1.フィードドキュメントを作成するで返された情報を使用して作成したフィードをアップロードできます。以下のサンプルコードで、フィードコンテンツをアップロードする1つの方法を示します。このサンプルコードに示されている原則は、他のプログラミング言語でのアプリケーション構築や、他のHttpClientライブラリ使用のガイドとしても使用できます。

UploadExampleクラスに示されているサンプルのuploadメソッドは、フィードコンテンツを1つ目の引数として受け取り、ステップ1で保存したurl値を2つ目の引数として受け取ります。

サンプルコード（Java）のアップロード

​x
// UploadExample.java
// これは以下での使用例です。Selling Partner API for Feedsのバージョン： 2021-06-30
import java.io.IOException;
import java.nio.charset.StandardCharsets;


import com.squareup.okhttp.MediaType;
import com.squareup.okhttp.OkHttpClient;
import com.squareup.okhttp.Request;
import com.squareup.okhttp.RequestBody;
import com.squareup.okhttp.Response;


/**
* コンテンツをアップロードする例。
 */
public class UploadExample {


  public static void main(String args[]) {
    String url = "<createFeedDocumentオペレーションのURL>";
    String content = "<フィードコンテンツ>";


    UploadExample obj = new UploadExample();
    obj.upload(content.getBytes(StandardCharsets.UTF_8), url);
}


  /**
* 指定したURLにコンテンツをアップロードします。
   *
* @param アップロードするコンテンツのソース
* @param url コンテンツをアップロードするURL
   */
  public void upload(byte[] source, String url) {
    OkHttpClient client = new OkHttpClient();


// contentTypeはcreateFeedDocumenオペレーションで入力したものと一致する必要があります。この例ではtext/xmlを使用していますが、contentTypeは選択したfeedType（text/plainやtext/csvなど）によって異なる場合があります。
    String contentType = String.format("text/xml; charset=%s", StandardCharsets.UTF_8);
    try {
      Request request = new Request.Builder()
        .url(url)
        .put(RequestBody.create(MediaType.parse(contentType), source))
        .build();


      Response response = client.newCall(request).execute();
      if (!response.isSuccessful()) {
        System.out.println(
          String.format("ドキュメントのアップロード呼び出しが失敗しました。レスポンスコード：%d、メッセージ：%s",
            response.code(), response.message()));
      }
    } catch (IOException e) {
      System.out.println(e.getMessage());
    }


}
}

ステップ5.フィード処理を確認する

フィードが以下のいずれかの端末状態に移行するまで、getFeedオペレーションを定期的に呼び出して、フィードの処理を確認します： DONE、CANCELLED、FATAL。フィードがDONE状態に移行したら、ステップ6.フィード処理レポート検索のための情報を取得するに進みます。

📘 フィードの処理には最大で8時間かかります

高負荷条件下では、フィードの処理に最大で8時間もかかることは珍しくありません。商品データフィードは順番に処理されます。最新のフィードは、前回のフィード送信が完了するまで処理システムのキューに入れられます。すべての商品が1つの商品フィードに含まれるのではなく、複数の商品フィードにそれぞれ少数の商品だけが含まれる場合は、処理が大幅に遅れる可能性があります。

  1. getFeedオペレーションを呼び出して、以下のパラメーターを渡します。

パスのパラメーター：

名前 説明 必須
feedId

フィードID。ステップ3. フィードを作成するで、createFeedオペレーションを呼び出した結果からこのIDを取得します。このIDは、出品者IDと組み合わせた場合のみ一意です。

タイプ：string

 はい

リクエストの例：

xxxxxxxxxx
GET https://sellingpartnerapi-na.amazon.com/feeds/2021-06-30/feeds/23492394

レスポンスの例

xxxxxxxxxx
{
  "payload":
{
    "processingEndTime":"2020-08-10T16:56:55+00:00",
    "processingStatus":"DONE",
    "marketplaceIds":[
      "ATVPDKIKX0DER"
    ],
    "feedId":"23492394",
    "feedType":"POST_PRODUCT_DATA",
    "createdTime":"2020-08-10T16:55:32+00:00",

  1. processingStatus属性の値を確認します。

    • processingStatusがIN_QUEUEまたはIN_PROGRESSの場合、フィードの処理はまだ完了していません。processingStatusが以下のいずれかの端末状態に達するまで、getFeedオペレーションを再試行します： DONE、CANCELLED、FATAL。
    • processingStatusがDONEの場合、フィード処理は完了です。ステップ6.フィード処理レポート検索のための情報を取得するに進みます。
    • processingStatusがCANCELLEDの場合、フィードは処理を開始する前にキャンセルされました。フィードを再度送信する場合は、ステップ1.フィードドキュメントを作成するからやり直してください。
    • processingStatusがFATALの場合、フィードは致命的なエラーのために停止されました。フィード内のオペレーションの一部またはすべてが正常に完了したか、すべてが完了しなかった可能性があります。Amazonがフィード処理レポートを生成する場合があります（ただしすべてに対してというわけではありません）。Amazonがレポートを生成する場合、正常に完了したフィードのフィード処理レポートとは異なる形式になることがあります。ステップ6.フィード処理レポート検索のための情報を取得するに進み、フィード処理レポートの検索を試行します。まれに、Amazonがフィードに関係のない理由でフィードを停止することがあります。修正すべきエラーがフィードに見つからなかった場合は、フィードを再度送信してみてください。

注： getFeedオペレーションは、過去90日以内に作成されたフィードリクエストの情報のみを提供します。

  1. urlとオプションのcompressionAlgorithm値を保存して、ステップ7.フィード処理レポートをダウンロードするで渡します。

ステップ7.フィード処理レポートをダウンロードする

前のステップで返された情報を使用して、フィード処理レポートをダウンロードできます。以下のJavaサンプルコードが役に立ちます。このJavaサンプルコードに示されている原則は、他のプログラミング言語でのアプリケーション構築のガイドとしても使用できます。

  1. サンプルコードの入力に以下を使用します。

    • 前のステップのurl値とオプションのcompressionAlgorithm値は、DownloadExampleクラスのdownloadメソッドのurlとcompressionAlgorithmパラメーターの引数です。

注： 開発者が責任を持って保存時に暗号化を常に維持してください。暗号化されていないフィード処理レポートのコンテンツは、一時的であっても、ディスクに保存しないでください。フィード処理レポートには機密情報が含まれる可能性があります。提供するサンプルコードは、この原則を示しています。

サンプルコード（Java）のダウンロード

xxxxxxxxxx
// DownloadExample.java
// これは、以下での使用例です。Selling Partner API for Reportsのバージョン： 2021-06-30
// およびSelling Partner API for Feedsのバージョン： 2021-06-30
import java.io.BufferedReader;
import java.io.Closeable;
import java.io.IOException;
import java.io.InputStream;
import java.io.InputStreamReader;
import java.nio.charset.Charset;
import java.util.zip.GZIPInputStream;


import com.squareup.okhttp.MediaType;
import com.squareup.okhttp.OkHttpClient;
import com.squareup.okhttp.Request;
import com.squareup.okhttp.Response;
import com.squareup.okhttp.ResponseBody;


/**
 * ドキュメントをダウンロードする例。
 */
public class DownloadExample {


  public static void main(String args[]) {
    String url = "<getFeedDocument/getReportDocumentレスポンスのURL>";
    String compressionAlgorithm = "<getFeedDocument/getReportDocumentレスポンスのcompressionAlgorithm>";


    DownloadExample obj = new DownloadExample();
    try {
      obj.download(url, compressionAlgorithm);
    } catch (IOException e) {
      //ここで例外を処理します。
    } catch (IllegalArgumentException e) {
      //ここで例外を処理します。
    }
}


  /**
   * 指定されたURLから取得したドキュメントをダウンロードし、必要に応じて解凍します。
   *
   * @param url ドキュメントを指すURL
   * @param compressionAlgorithm ドキュメントに使用されたcompressionAlgorithm
   *  @throws IOException  レスポンスの読み取り中にエラーが発生した場合
   * @throws IllegalArgumentException  その文字セットがない場合
   */
  public void download(String url, String compressionAlgorithm) throws IOException, IllegalArgumentException {
    OkHttpClient httpclient = new OkHttpClient();
    Request request = new Request.Builder()
      .url(url)
      .get()
      .build();


    Response response = httpclient.newCall(request).execute();
    if (!response.isSuccessful()) {
      System.out.println(
        String.format("コンテンツをダウンロードする呼び出しは失敗しました。レスポンスコード：%d、メッセージ：%s",
          response.code(), response.message()));
      return;
    }


    try (ResponseBody responseBody = response.body()) {
      MediaType mediaType = MediaType.parse(response.header("Content-Type"));
      Charset charset = mediaType.charset();
      if (charset == null) {
        throw new IllegalArgumentException(String.format(
          "'%s'の文字セットを解析できませんでした", mediaType.toString()));
      }


      Closeable closeThis = null;
      try {
        InputStream inputStream = responseBody.byteStream();
        closeThis = inputStream;


        if ("GZIP".equals(compressionAlgorithm)) {
          inputStream = new GZIPInputStream(inputStream);
          closeThis = inputStream;
        }


        // この例では、ダウンロードコンテンツのContent-Typeヘッダーなどに文字セットがあることを前提としています。
        // text/plain; charset=UTF-8
        if ("text".equals(mediaType.type()) && "plain".equals(mediaType.subtype())) {
          InputStreamReader inputStreamReader = new InputStreamReader(inputStream, charset);
          closeThis = inputStreamReader;


          BufferedReader reader = new BufferedReader(inputStreamReader);
          closeThis = reader;


          String line;
          do {
            line = reader.readLine();
            // 1行ずつ処理します。
          } while (line != null);
        } else {
          //ここでバイナリデータ/その他のメディアタイプのコンテンツを処理します。
        }
      } finally {
        if (closeThis != null) {
          closeThis.close();
        }
      }
    }
}
}

ステップ8.フィード処理レポートのエラーをチェックする

フィード処理中に生成されたエラーがないか、フィード処理レポートを確認します。エラーがない場合は、フィードの送信が完了しています。エラーがある場合は、それを修正して、修正済みフィードを送信します。（ステップ1.フィードをアップロードできます。フィード処理レポートにエラーがなくなるまで、この処理を繰り返します。
